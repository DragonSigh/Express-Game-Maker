Maps

Maps are essential to your games. A map contains the game's environment and events.

In order to create a map, first click on the button.

Next, click the button to add a new map. Enter the map's name and click OK.

Once the map is created you can modify its settings from its menu

  • Resize the map.
  • Resize the map's grid.
  • Change the map's gravity.
  • Add effects such as background music and screen tint.
  • Toggles the opacity of seperate layers, useful for general mapping.
  • Shows the map's grid which can be an useful visual aid.
  • Snaps tiles to a grid. Useful for tile based mapping.

In order to draw your map, you'll need a tileset ready. If you don't know how to make a tileset, check out the previous "Tilesets" tutorial.

Once you create your tileset, select your tile from Tiles explorer. Make sure is checked and eraser is not checked. Click on the map to place your tile. If "Snap to Grid" is on, you can click and drag your mouse to paint.

A map can have unlimited layers. Layers can be used differently for different genres. For example, in a platformer game, everything you can interact with is usually on the top most layer, the rest is scenery. In an RPG game, layers are usually used to show tiles above or below player.

You can only edit the layer that is selected. If you have a tile or an event that is on the previous layer, you must select the previous layer and then select the tile.

To edit your tiles' rotation, scaling and opacity, you'll need to select . Double click the tile to bring up the tile settings dialog. Note: If the event view is checked, then you can only edit events.'

When mapping for top-down RPG maps, there are some guides you should follow:

  1. Setup your layers.
    1. To keep it basic, set two layers for below events, one event layer, and two above layers.
    2. In non-grid mode, you can layer tiles on top of each other in a single layer. Useful for adding detail.
  2. Design your map.
    1. Consider collision sizes, passages and walkways.
    2. Do not place collidable objects on pathways such as rocks. Placing things like wells, trees and similar large objects for architecture is fine.
    3. Consider what needs to be always above events when mapping. If you make a mistake, go to cursor mode, right click and either move the tile down or up a layer.
  3. Test your map and correct any problems with layering or collision.
  4. Add extra details, the little things do count.

  5. Test again to make sure everything is where it's suppose to be and in its correct layers.
